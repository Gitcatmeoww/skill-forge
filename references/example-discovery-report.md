## Synthesis

### Patterns to adopt
1. **Risk scoring** (migrate-guard) — assigning severity levels to detected patterns helps users prioritize. Works because it turns a wall of warnings into actionable triage.
2. **Hybrid deterministic + LLM review** (migration-lint) — use scripts for mechanical checks, LLM for semantic understanding. Plays to each tool's strengths.
3. **Safe migration checklist** (db-migrate-check) — giving users a concrete verification checklist improves confidence in the review.

### Anti-patterns to avoid
1. **Piping SQL to database CLIs** (sql-reviewer) — executing migration content directly is a critical security risk. Review should be read-only.
2. **Overly long SKILL.md** (sql-reviewer at 420 lines) — instruction fatigue causes Claude to skip or misweight later sections.
3. **Vague trigger descriptions** (sql-reviewer) — leads to false activations on non-migration SQL work.

### Security concerns to address
1. Unvalidated file path inputs in shell scripts (migrate-guard) — severity: warning
2. Direct SQL execution without sanitization (sql-reviewer) — severity: critical
3. Unpinned pip install at runtime (migration-lint) — severity: warning

### Gaps to fill
1. **No data migration support** — all 4 skills only handle schema changes. Backfills, data transforms, and seed data are uncovered.
2. **No rollback guidance** — skills flag problems but never suggest rollback strategies or safe alternatives.
3. **No ORM migration support** — all skills assume raw SQL. Alembic, ActiveRecord, Prisma, and Knex migrations are ignored.
4. **No dry-run workflow** — none enforce or suggest a dry-run step before applying migrations.

---

## Recommendation

The new skill should adopt the risk-scoring and hybrid-review patterns, expand scope to cover data migrations and ORM frameworks, and add a mandatory dry-run step. Critical differentiator: rollback guidance for every flagged issue (not just "this is dangerous" but "here's how to make it safe"). Security posture should be read-only — never execute migration SQL, only analyze it.
